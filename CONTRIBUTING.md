# Contributing to MediQuery Pro

First off, thank you for considering contributing to MediQuery Pro! It's people like you that make MediQuery Pro such a great tool.

## How Can I Contribute?

### Reporting Bugs
*   Check if the bug has already been reported in the Issues.
*   If not, open a new issue with a clear title and description.
*   Include steps to reproduce and your environment details.

### Suggesting Enhancements
*   Open an issue to discuss your idea before implementing it.
*   Focus on how it benefits the medical or AI community.

### Pull Requests
1.  Fork the repo and create your branch from `main`.
2.  If you've added code that should be tested, add tests.
3.  Ensure the test suite passes.
4.  Make sure your code lints.
5.  Issue that pull request!

## Coding Standards
*   Follow PEP 8 for Python.
*   Use type hints where possible.
*   Document new functions and classes.
*   Keep the UI clean and accessible.

## Community
Join our discussions and help us build the world's best health retrieval system!
